Participate!

Report Bugs

You can help a lot by using Selenium Grid and reporting problems that you find. You can file any bug you discover by email or using our bug tracking tool.

Join the Community

Hang out in our User Forum. Help new users to get started and veterans to chase a nasty bug. Feel free to also share a new trick or start discussing how we can improve Selenium Grid for the next release...

Spread the Word

Let people know about Selenium Grid. Share your knowledge with your co-workers, your friends or your local Ruby/Java/XP/Pattern group. Why not even blog about it, and tell the whole world!

Express Yourself

If you have a talent for graphics or web design we can use some help. We have plenty of ideas for logos or little cartoons related to Selenium Grid but none of us are up to the task. We also welcome alternative designs for this website (we favor simple and to the point, but we also do not mind esthetics and fun). Please contact us by email if you feel that you can contribute in that area!

Write Kick-Ass Documentation

Help us improve the existing documentation and create new content. Clear and well written documentation harnessing collective knowledge is a terrific asset for any community. Report English mistakes on the website, create a new page on the wiki, write a blog post or publish an article.

Hack Some Code

We welcome new contributors. The project is still at a very early stage and you can have significant impact. We only ask that you write beautiful code and write tests ;-)

Shop around in our Card Wall and Project Backlog to find something that you want to do. Let us know that you are working on this particular task, or update the page with your initials/email next to the task. This way we can keep track of who is working on what. In any case you are better off if you send us an email first as it is hard to understand what a card actually means by just reading its description on the card wall.

Here is the best way to start hacking:

  1. Install Java 5+ JDK and make sure it is in your PATH: 
    $ java -version
java version "1.6.0"
Java(TM) SE Runtime Environment (build 1.6.0-b105)
Java HotSpot(TM) Server VM (build 1.6.0-b105, mixed mode)
  2. Install Ant 1.7.x and make sure it is in your PATH: 
    $ ant -version
Apache Ant version 1.7.0 compiled on December 13 2006
  3. svn co https://svn.openqa.org/svn/selenium-grid/trunk <project root>

  4. cd <project root>

  5. Run the build, to make sure that it passes on your box: 
    $ ant dist
  6. Hack some cool features and do not forget to write some tests. Write them first if possible.

  7. Run the build again, make sure that it still passes. 
    $ ant dist
  8. Create a patch: 
    $ svn diff > patch_to_fix_this_and_that.diff
  9. Send us the patch by email
